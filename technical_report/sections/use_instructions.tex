\subsubsection{Installation Procedure}

\paragraph{1. Retrieve Source Code} The system source code can be obtained from the project repository. This command downloads the project files and navigates into the project directory.

\begin{commandbox}
git clone \textless repository-url\textgreater \\
cd project-directory
\end{commandbox}

\paragraph{2. Install Backend Dependencies} The backend server dependencies must be installed to enable the backend application server and scraping capabilities. 

\begin{commandbox}
cd backend \\
npm install
\end{commandbox}

\paragraph{3. Install Frontend Dependencies} The frontend dependencies must be installed to run the React user interface utilising the required packages. 

\begin{commandbox}
cd ../frontend \\
npm install
\end{commandbox}

\paragraph{4. Configure Environment Variables} The backend functionality requires environment variables to be configured to provide access to the database and admin account settings. A \texttt{.env} file must be created in the backend directory containing the configuration settings. 

\begin{commandbox}
DATABASE\_URL=\textless postgresql\_database\_url\textgreater \\
ADMIN\_PASSWORD\_HASH=\textless hashed\_admin\_password\textgreater \\
PORT=\textless selected\_port\textgreater
\end{commandbox}
The port can be chosen by the user, however, will default to 5001 if not included. The remaining values will be defined during the database configuration and initial account setup, described in Section~\ref{sec:deployment} and Section~\ref{sec:launching}.

\subsection{Deployment (5\%)}
\label{sec:deployment}
% Local machine deployment, database setup, VMs/Containers/Cloud.

Once installed, the application must be configured and deployed before users can interact with the system. 

\paragraph{1. Create PostgreSQL Database} The system requires a PostgreSQL database to store the data collected from UCAS and individual institution websites in a structured format. 

\begin{commandbox}
createdb spio\_course\_db
\end{commandbox}

The database name must correspond with the value specified in the \texttt{DATABASE\_URL} environment variable configured. 

\paragraph{2. Generate Administrator Password Hash} An account is required to access the frontend due to the protected routes. The administrator account must be created, with the password stored in a secure hashed format. A script is provided to accept the desired admin password, and outputs the bcrypt password hash. This ensures that unauthorised users that could possibly access the website cannot access the protected data. 

\begin{commandbox}
npx ts-node backend/src/scripts/createPassword.ts \textless password\textgreater
\end{commandbox}

The output hashed password value must be inserted as the \texttt{ADMIN\_PASSWORD\_HASH} environment variable.

\paragraph{4. Initialise Prisma Database Schema} The Prisma ORM manages the backend database access and schema. These commands generate the Prisma client and apply the required database migrations.

\begin{commandbox}
cd backend \\
npx prisma generate \\
npx prisma migrate deploy
\end{commandbox}

This creates the database tables for the collected data to be inserted into and generate client for the backend server to interact with the database. 

\paragraph{5. Start Application Services} Once the database has been created, the backend and frontend can be started to provide the system interface.

\begin{commandbox}
cd backend \\
npm run build \\
npm run start
\end{commandbox}

This command initiates the backend API server to receive the frontend requests, and the frontend server providing web browser access to the system. 

\subsection{Launching (5\%)}
\label{sec:launching}
% User account creation, login, different roles, first-time setup.

Once the application services have been started during deployment, the system can be accessed through a web browser. The following steps describe how to access and perform the initial setup required to use the platform. 

\paragraph{1. Access Web Application}
During deployment, the frontend and backend services are initialised, allowing the system to be accessed through a browser. 

\begin{commandbox}
http://localhost:PORT
\end{commandbox}

Navigating to this URL loads the login page, which when authenticated provides access to the protected routes. Any attempt to navigate to a sub-page will redirect to this login page until authenticated. 

\paragraph{2. Authenticate with Administrator Account}
The administrator account was initialised during deployment due to its required backend configuration. On login, the system authenticates the password attempt against the hash, once verified, provides access to the protected routes. 

\begin{commandbox}
Username: admin \\
Password: \textless ADMIN\_PASSWORD\textgreater
\end{commandbox}

Once authenticated, the users is redirected to the system dashboard, or specific sub-page routed to in the URL. Access to permitted to the protected routes, including course data pages, scraping tools and export functionality. Authentication is maintained through a session-based token, ensuring only authorised users can access the system. 

\paragraph{3. Initial Data Scrape}
On launch, the database is empty, therefore the competitor course data must be collected before the features become available. From the dashboard page, the user can initialise the collection process by clicking the \texttt{Quick Fetch} button. The dashboard interface updates to indicate the stage of the process, in progress or completed.
